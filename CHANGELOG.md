# 更新日志  

## Version 4:  
### 创建者：周宇扬  
###更新时间：  
2026.6.16 22：00  
### 更新内容：  
1.大幅修改了路径规划部分代码，抛弃了A*算法（程序名还叫A*是因为避免修改启动命令和setup等部分，实际上只有文件名还叫astar,内容和A*毫无关系，实属是A*风味了）。   
2.小幅修改了控制部分，稍微调整了一下纯跟踪的代码   
####开发过程：
1.本来想继续沿用A*算法的，但是无论如何尝试都没办法使用一个全局路径规划使得车辆乖乖留在赛道内而不切弯出去，遂放弃）   
2.接着想使用车队给的感知节点实现锥桶位置和颜色的感知，以方便建图规划，但是不知道什么原因，节点发布的锥桶颜色有红有蓝，但是到了实际调用只剩蓝色锥桶，导致车辆只会原地转圈，尝试多种修复办法均无效，遂放弃  
3.如今只能使用纯雷达的方法了。其原理是将车辆前方一定距离的锥桶按照车辆左右分为两组，然后按照到车辆的纵向距离进行两两匹配，找出中心点即为赛道中线。然而，在运行过程中雷达对左右两侧的锥桶检测非常不对称，所以加了很多代码提高鲁棒性。最终效果也很一般，能够正常通过弯道还是小概率事件，一致性很差。  
###存在问题：  
1.纯雷达检测相当不稳定  
2.路径规划部分的众多参数需要调整，纯跟踪的前瞻距离也要适当调整  
###注意：  
如果你的雷达感知数据里，左右锥桶比较对称，可以考虑使用“开发过程”中提到的第二种思路，应该是比较好的方法
